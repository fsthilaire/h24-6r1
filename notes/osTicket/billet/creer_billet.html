Créer un billet

Cette procédure explique comment un agent peut créer un billet pour un utilisateur.

Procédure​

  1. Connectez-vous au panneau de l'agent.

  2. Cliquez sur l'onglet Ticket.

  3. Appuyez sur le bouton Nouveau ticket.

  4. Inscrivez les 3 lettres du prénom. nom ou courriel de l'utilisateur dans le champ Rechercher.

  5. Cliquez sur l'utilisateur dans la liste.

  6. Appuyez sur le bouton Poursuivre pour assigner l'utilisateur au billet.

  7. Dans la section Utilisateur et collaborateurs, il faut mettre l'information initiale du billet. En jaune, ce sont les champs obligatoires et en orange les champs optionnels.

    Description des champs

    • Source du ticket : Détermine le point de contact initial avec l'utilisateur.

      • Utilisez la valeur Autre si c'est en personne.

    • Rubrique d'aide : Permet d'indiquer si le billet est pour une régler un problème ou pour une demande particulière.

    • Date d'échéance : (Optionnel) Permet d'indiquer une date limite pour compléter la demande.

    • Assigné à : (Optionnel) Permet d'indiquer le responsable du billet.

      • Généralement, il faut assigner la demande à soi-même.

      • Il est possible de l'assigner à un autre membre de l'équipe.

      • Si le billet n'est pas assigné à une personne, le billet sera attribué au prochain agent disponible.

  8. Dans la section Détail du billet, il faut mettre l'information du billet. En jaune, ce sont les champs obligatoires et en orange les champs optionnels.

    Description des champs
    • Résumé : Il faut décrire la demande ou le problème en quelques mots.
      • Ce champ est utilisé pour la recherche dans la liste des billets.
    • Description détaillée : Il faut décrire la demande ou le problème en détail.

      • Il s'agit d'un éditeur WYSIWYG. Il est possible d'incorporer des images.

      • Il est possible d'ajouter des fichiers.

    • Priorité : (Optionnel) : Permet d'indiquer le niveau de priorisation du billet pour l'équipe.
    Important

    Tout le texte inscrit dans cette section est visible à l'utilisateur.

  9. La section Réponse permet de communiquer avec l'utilisateur. Cette section est optionnelle lors de l'ouverture du billet.

    Description des champs
    • Réponse initiale : Il faut décrire la demande ou le problème en détail.
      • Il s'agit d'un éditeur WYSIWYG. Il est possible d'incorporer des images.
      • Il est possible d'ajouter des fichiers.
    • État du ticket : Permet d'indiquer l'état du ticket.
      • Ouverts : La demande ou le problème est en cours de résolution.
      • Resolved : La demande ou le problème a été résolu.
      • Fermé : La demande ou le problème n'a pas été résolu et il n'y aura pas d'autres démarches pour la résolution.
    Important

    Tout le texte inscrit dans cette section est visible à l'utilisateur.

  10. La section Note interne permet d'inscrire de l'information sur le billet pour le reste de l'équipe. Cette section est optionnelle.

    Description du champ
    • Note interne : Il faut décrire la demande ou le problème en détail.
      • Il s'agit d'un éditeur WYSIWYG. Il est possible d'incorporer des images.
    Important

    Tout le texte inscrit dans cette section n'est pas visible à l'utilisateur.

  11. Appuyez sur le bouton Ouverts à la fin du formulaire pour créer le billet.
