Guide d'utilisation

Le guide d'utilisation ou guide d'utilisateur est un document qui explique le fonctionnement du logiciel.

Ce type de document peut se séparer en 2 parties :

  • Explication d'une tâche, une action ou d'une procédure du logiciel
  • Explication de la structure du logiciel
    • Menu
    • Boite à outils
    • Type d'icône
    • Logique des menus
    • Description des champs d'un formulaire
    • Lexique de l'application

Voici les grandes lignes de ce qui permet d'avoir un guide utilisateur bien conçu.

Structurer le guide et les articles​

Il est important de bien structurer le guide.

Par exemple, regrouper les tâches et les explications par modules.

Utiliser une table des matières pour naviguer facilement dans le document.

Permettre la recherche :

  • Indexation des sites web
  • PDF qui est en mesure de rechercher dans le texte

Utiliser des sections dans les articles pour séparer les éléments importants.

Cas d'utilisation​

L'explication des tâches devrait se rapprocher au cas d'utilisation du logiciel.

Éviter de regrouper plusieurs cas dans le même article.

Ex. : Il faut un document pour la connexion et un document pour l'inscription.

Pour se connecter, veuillez inscrire votre nom d'utilisateur.

  • Si vous n'avez pas de compte, cliquez sur instruction.
  • Entrez les informations.

Entrez votre mot de passe et appuyez sur le bouton Connecter.

Titre de l'article​

Le titre doit être très significatif et permet de savoir ce que la tâche va faire.

Mise en contexte​

Pour l'explication d'une tâche ou d'une procédure, il est important de faire une mise en contexte.

Elle permet d'expliquer sommairement la tâche et sa finalité.

Prérequis​

Les prérequis permettent d'indiquer les éléments qui sont nécessaires de faire ou d'avoir pour être en mesure de faire la procédure.

  • Activer un paramètre dans l'application
    • Ex. : Pour recevoir des notifications par courriel, il faut activer l'envoi dans son profil utilisateur.
  • Avoir un droit ou privilège spécifique
    • Ex. : Seulement les administrateurs peuvent faire la tâche.
  • Effectuer une sous-tâche
    • Ex. : Pour se connecter au système, il faut avoir créé son compte.

Lister les étapes​

Pour lister les étapes, il faut favoriser l'utilisation de puce numérotée.

Le numéro permet d'indiquer clairement et sans ambiguïté une étape précise dans l'article.

Éviter les gros paragraphes.

Explication détaillée

Pour les explications détaillées, il est préférable d'utiliser un encadré ou un élément visuel pour facilement repérer que c'est de l'information additionnelle et non dans la procédure.

Il est important d'expliquer la procédure de bout en bout. Il ne faut pas ignorer des étapes.

Image​

Il faut utiliser des images pour appuyer les étapes.

Si une image est utilisée pour plusieurs étapes, il est important de mettre la séquence des actions en évidence.

Il peut être plus facile de suivre une image par étape.

L'image devrait être dans la même langue des éléments mentionnés.

  • Ex : Menu Fichier, mais l'image monte File.

Mise en évidence​

Mettre en évidence les mots importants qui représente le texte des boutons, onglets, champs...

L'utilisation de gras est privilégiée.

Tableau​

L'utilisation d'un tableau permet de structurer de l'information.

Le tableau peut être utilisé pour lister les champs d'un formulaire.

Ex. :

ChampObligatoireExplication
Catégorie de produitOuiPermet de spécifier la catégorie du produit pour faciliter le classement et la recherche
NoteNonPermet d'ajouter des notes internes pour l'équipe de vente.

Limite​

Il peut avoir des limites logicielles dans certaines fonctionnalités en fonction. Il peut être pertinent de les inclure dans la tâche.

Ex. : Un employé peut avoir uniquement 3 numéros de téléphone dans sa fiche d'employé.

Référence​

Utilisez des liens pour référer une section de l'article.

Utilisez des liens pour référer un autre article.

Adapté à l'utilisateur​

Il faut que le guide soit adapté à l'utilisateur.

La langue utilisée est celle de l'utilisateur.

Le niveau de compétence de l'utilisateur peut déterminer si l'utilisation de raccourci ou de commandes est pertinente.

Version de l'application​

Il est important de spécifier la version de l'application, lorsqu'il existe plusieurs versions.

La procédure peut être différente selon la version.

Les images qui représentent l'application ne sont plus identiques.

Date du document​

La date du document est un indicateur pour l'utilisateur si le document est récent ou non.

Un ancien document peut indiquer à l'utilisateur qu'il n'utilise peut-être pas la version la plus récente.

La date peut être affichée publiquement ou dans les données internes de l'article.

Auteur​

Il est important de connaitre l'auteur d'un document ou d'un article.

Pour les guides publics, il n'est pas conseillé d'afficher publiquement cette information. L'utilisateur pourrait essayer de contacter directement l'auteur pour avoir de l'aide.

Pour les guides internes, cette information peut être visible à tous.
